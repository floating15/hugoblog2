Git - Intro

2019/07/23

1. Git intro

Git 是一个分布式版本控制软件。版本控制的方式一共经历过四个发展阶段：

  1. 文件管理
  2. 本地管理软件
  3. 集中式管理软件，以 SVN 为代表
  4. 分布式管理软件，以 Git 为代表

在 Git 中，用户本地与云端服务器中都存在版本仓库。当用户打算更新版本时，需要将新版本文件推送到本地仓库，再推送到云端仓库。用户也可以多次将新版本推送到本地仓库，再统一推送到云端仓库。因此所云端服务器崩溃，不影响用户的版本控制。

在本地安装 Git 后，可以在本地进行版本控制；若要将版本代码上传至云端仓库，需要链接 Github, Gitlab 账户等。

2. Git 三大分区

Git 有三种状态，你的文件可能处于其中之一： 已暂存（staged）、已修改（modified） 和已提交（committed） 。

  1. 已修改（modified）表示修改了文件，但还没保存到数据库中。
  2. 已暂存（staged）表示对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中。
  3. 已提交（committed）表示数据已经安全地保存在本地数据库中。

这会让我们的 Git 项目拥有三个阶段：工作区、暂存区、版本库。基本的 Git 工作流程如下：

  1. 在工作区中修改文件，文件状态变为 modified。
  2. 将你想要下次提交的更改选择性地暂存，这样只会将更改的部分添加到暂存区，文件状态变为 staged。
  3. 提交更新，找到暂存区的文件，将快照永久性存储到 Git 目录，文件状态变为 committed。

3. Git 命令基础

利用 Git 进行版本控制，大致有以下几步：

3.1. 进入要管理的目录

进入工作区：

(base) wanghaoming@localhost ~ % cd ~/Documents/MATLAB 
(base) wanghaoming@localhost MATLAB %

3.2. 初始化 Git

(base) wanghaoming@localhost MATLAB % git init
hint: Using 'master' as the name for the initial branch. This default branch name
hint: is subject to change. To configure the initial branch name to use in all
hint: of your new repositories, which will suppress this warning, call:
hint: 
hint: 	git config --global init.defaultBranch <name>
hint: 
hint: Names commonly chosen instead of 'master' are 'main', 'trunk' and
hint: 'development'. The just-created branch can be renamed via this command:
hint: 
hint: 	git branch -m <name>
Initialized empty Git repository in /Users/wanghaoming/Documents/MATLAB/.git/

执行完该命令后，MATLAB 目录内会生成.git隐藏目录，Git 所有的配置与版本信息都会储存到.git目录中。

3.3. 管理

检测当前目录下的文件状态：

(base) wanghaoming@localhost MATLAB % git status
On branch master

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	AHPcal.m
	untitled.fig
	untitled.m
	untitled.pdf

nothing added to commit but untracked files present (use "git add" to track)

将要管理的文件添加到暂存区：

(base) wanghaoming@localhost MATLAB % git add AHPcal.m 
(base) wanghaoming@localhost MATLAB % git status
On branch master

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
	new file:   AHPcal.m

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	untitled.fig
	untitled.m
	untitled.pdf

可见文件AHPcal.m已被 git 管理；若要继续管理所有未被管理的文件可以使用.:

(base) wanghaoming@localhost MATLAB % git add .
(base) wanghaoming@localhost MATLAB % git status
On branch master

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
	new file:   AHPcal.m
	new file:   untitled.fig
	new file:   untitled.m
	new file:   untitled.pdf

指定选项-A可以一次性管理目录内所有文件。

3.4. 个人信息配置

第一次使用 Git 时，在进行下一步提交到版本库之前，需要先对 Git 进行配置（用户名，邮箱）

git config --global user.email "wanghaoming17@163.com"
git config --global user.name "wanghaoming17"

Git 在 commit 时会自动记录该版本由谁生成，可以在 git log 中查看。

3.6. 回滚

git reset hard 版本号

将版本库中的文件添加到工作区中，即要恢复到之前的某一版本，只需要将 git log 中显示的 commit 号粘贴至 git reset --hard 后即可：

(base) wanghaoming@localhost MATLAB % git reset --hard bf75a8c80fe17c8c07b554858451ed162bc64a0f 
HEAD is now at bf75a8c v1

此时 AHPcal.m 文件中已经没有之前的修改信息了：

此时查看版本日志会发现 v2 版本消失了：

(base) wanghaoming@localhost MATLAB % git log 
commit bf75a8c80fe17c8c07b554858451ed162bc64a0f (HEAD -> master)
Author: wanghaoming17 <wanghaoming17@163.com>
Date:   Fri Mar 4 08:48:15 2022 +0800

    v1

若要再回滚到 v2 版本则通过 git reflog 查看版本日志：

(base) wanghaoming@localhost MATLAB % git reflog
bf75a8c (HEAD -> master) HEAD@{0}: reset: moving to bf75a8c80fe17c8c07b554858451ed162bc64a0f
742bb77 HEAD@{1}: commit: v2
bf75a8c (HEAD -> master) HEAD@{2}: commit (initial): v1

reset v2的版本号即可：

(base) wanghaoming@localhost MATLAB % git reset --hard 742bb77
HEAD is now at 742bb77 v2

此时 AHPcal.m 表头的 Git test 又出现了：

git checkout

4. 总结：

  • git init: 初始化，让 git 管理当前目录
  • git status：检测当前目录下文件的状态
  • git add：管理未被管理的文件
  • git commit：生成版本
  • git log：查看已经生成的版本
  • git reflog
  • git reset --hard 版本号：版本库回滚至工作区未修改状态
  • git reset HEAD <file name>: 暂存区回滚至工作区修改状态
  • git checkout -- <file name>：工作区修改状态回滚至工作区未修改状态
